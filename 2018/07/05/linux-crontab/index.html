crontab定时任务不执行，单独运行sh生效

原因1：环境变量未生效，如Java的环境变量

解决办法：添加一句source /etc/profile

原因2：可执行文件缺少参数

解决办法：统一将启动程序写成脚本，然后由 monitor.sh 调用这个 start.sh

比如启动nexus程序的脚本start.sh
